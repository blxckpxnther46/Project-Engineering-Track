Checkpoint 1: Token Generation ✅
  Location: controllers/authController.js (lines 29-34)
  Bug Found: 
    • Payload missing email and role
    • Secret hardcoded as 'mysecretkey'
    • No expiration set
  Impact: Forever-valid tokens, exposed secrets in source code

Checkpoint 2: Middleware Header Extraction ✅
  Location: middleware/authMiddleware.js (line 4)
  Bug Found:
    • Reading from req.headers.token (custom header)
    • Not reading from req.headers.authorization (standard)
    • No Bearer prefix stripping
  Impact: Non-standard header format, incompatible with clients

Checkpoint 3: Verification Error Handling ✅ [CRITICAL]
  Location: middleware/authMiddleware.js (lines 13-19)
  Bug Found:
    • next() called in catch block
    • Bad tokens pass through to route handlers
  Impact: COMPLETE AUTHENTICATION BYPASS

Checkpoint 4: Route Protection Coverage ✅
  Location: routes/taskRoutes.js, routes/adminRoutes.js
  Bug Found:
    • GET /api/tasks/:id - NO authMiddleware
    • DELETE /api/tasks/:id - NO authMiddleware
    • GET /api/admin/users - NO authMiddleware or role check
    • DELETE /api/admin/users/:id - NO authMiddleware or role check
  Impact: Unauthorized access to sensitive endpoints

Checkpoint 5: Error Response Consistency ✅
  Issue: Inconsistent error handling and response formats
  Impact: Unreliable security signals for clients
